
	eraOCG2_V0.065用 追加キャラ&一斉召喚・パンツパッチ

★導入方法
	追加キャラ	→	[追加キャラ]フォルダ内のファイルを、CSV>CHARA>適切なキャラクター番号のフォルダにコピーしてください。
					キャラクターの追加状況によっては、ファイル内の番号項目を変更する必要があります。
					Chara151は追加衣装セットを使用しているため、導入の際はCLOTHES.ERBも併せて導入してください。

	パッチ	→	導入したい機能に合わせて、ERBファイルをコピーし既存のファイルに上書きしてください。

				改変キャラ	→	ERB>OBJ>CLOTHES.ERB
								改変キャラはCSV>CHARA>適切なキャラクター番号のフォルダにコピー

				一斉召喚パッチ	→	ERB>DIMS>DIM.ERH
									ERB>イベント関連>EVENTTURNEND.ERB
									ERB>召喚システムとその他作者オリジナル(COMF 450)>COMF450 モンスター召喚.ERB

				パンツキャラパッチ	→	ERB>コマンド関連>COMMANDS>499まで>COMF410.ERB

★パッチ内容

	★改変キャラ
	個人的主観によるキャラ改変が含まれています。
	Chara143,Chara147→専用の衣装セットがあるのに適用されていなかったので修正。意図してのものだったらごめんなさい。
	Chara40→略称ラビュ姫よりラビュリンスの方がよくない？短い方がいい？カテゴリ名と丸被りは嫌？そう・・・

	★一斉召喚パッチ
	前日に召喚したモンスターを一度に全部召喚することができるようになります。
	モンスター召喚[450]を選択した際に、前日にモンスターを召喚していれば

		[3] - 昨日のモンスターを召喚

	が表示されるようになります。
	選択すると、前日に召喚を行ったモンスターの内、同じ部屋にいないモンスター全てを召喚します。
	すでに自宅にいるモンスターも全て召喚します。（全員が今いる部屋に来ます）
	前日に自宅にいたモンスターでも、召喚を行ってないモンスターであれば呼び出しません。（同居している場合などが該当）
	※モンスター召喚[450]を選択するたびに対象モンスターのリストアップを行います。前日に召喚したモンスターが多いと、処理が重くなる可能性があります。
	※前日に召喚したモンスターの情報はセーブされません。SAVEDATA属性をつければセーブ可能そうですが、データの互換性などがよくわからないので自己責任でお願いします。

	★パンツキャラパッチ
	拾えるパンツの持ち主が、現在自宅にいるランダムなキャラになります。
	※一斉召喚パッチで導入した変数を利用し、昨日と今日の召喚情報を利用してパンツを落としたキャラを決定するパターンも用意してあります。
	　そちらを利用する場合は、一斉召喚パッチも併せて導入した上で、COMF410.ERBの該当部分を変更してもらう必要があります。（27～29行目をコメントアウトし、7行目・32～35行目をコメントでなくする）
	※その部屋に行ったことのないキャラでもパンツを落としている場合があります。気にするな。

★備考
	パッチの追加部分については、<;new ～ ;newここまで>で囲われています。
	同様に削除部分（改変部分）については、<;original ～ ;originalここまで>で囲われています。
	改変者以外でも既存からの改変点がわかりやすいようつけたものですので、確認したら適宜削除してください。
	（CSVは1行修正なので付けていません。統一性がない。）
	
	COMF410とCOMF450ではキャラのリストアップに配列を使っています。
	現状の配列の大きさは500なので、キャラを増やしまくる場合は適宜変更してください。

	変数名は適当です。一応被りがないことは確認しています。
	ただ、COMF410とCOMF450で使用している変数<召喚リスト_～>は、COMF450内に類似名の変数を確認しています。
	可読性に問題があれば変更してください。
